# Микростоковая фотография — проект Andrey Kotko

## О проекте

Портфолио микросток-иллюстратора. Основные тематики: цветочные векторные
иллюстрации, паттерны, свадебные приглашения, ботанические элементы.
Платформы: Shutterstock, iStock, Adobe Stock.
Цель: $1000/мес с продаж на стоках.

## Язык

Общение на русском. Файлы и метаданные на английском.

## Текущий этап

Именование файлов и простановка метаданных в MicrostockPlus для загрузки на стоки.
Чеклист прогресса: `progress_naming.md`

## Основные рабочие процессы

### 1. Уточнения ключевых слов для iStock

Самая частая задача. Когда пользователь говорит "проставь" / "проставь уточнения":

1. Проверь что модалка открыта (`.dialog-window` с чекбоксами)
2. Прочитай `references/reject-labels.md` из скилла `istock-refinements`
3. Проставь ВСЁ кроме reject-листа — пользователь ожидает максимум (40-80 галочек)
4. Сообщи сколько проставлено

**Ключевой принцип:** отклоняй только явный мусор (омонимы-певцы, спорт, политика,
баги iStock). Всё остальное — проставляй.

Подробнее: `.claude/skills/istock-refinements/SKILL.md`

### 2. Установка страны (Spain) для релизов

Для каждого файла нужно установить Spain в выпадающем списке страны.

### 3. Навигация по файлам

- `#toggle-next` / `#toggle-prev` — следующий/предыдущий файл
- `#save-metadata` — сохранить метаданные
- `.edit_keyword_terms` — открыть модалку уточнений

## Технические заметки

### Claude in Chrome MCP

Все действия на microstock.plus выполняются через `javascript_tool`.
Перед началом работы всегда:
1. Вызови `tabs_context_mcp` для получения `tabId`
2. Убедись что открыта правильная страница

### Частые ошибки

- `await` нельзя на верхнем уровне в `javascript_tool` — оборачивай в `(async () => {})()`
- `window`-функции теряются при перезагрузке — переинжектируй
- Автоцикл нажимает не те кнопки — используй строго `getElementById`
- Модалка грузится до 10 сек — жди, не кликай раньше

### Предпочтения пользователя

- Максимум галочек — не экономь, проставляй всё что не мусор
- Пошаговый режим надёжнее автоцикла
- Короткие ответы — "45 ✓ Сохраняй!" лучше чем длинные объяснения
- Не спрашивай лишнего — если тема озвучена, работай
